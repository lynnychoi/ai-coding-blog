# 히어로 섹션 하나 만드는 데 하루 다 썼다

## 이게 맞나 싶었는데

오늘은 블로그 첫 화면 디자인 작업을 했다. 코드 한 줄 없이 기획서만 쓰던 사람이 CSS 애니메이션을 만지고 있으니 이게 맞나 싶었는데 — 일단 됐다.

> **히어로 섹션?** 웹사이트 첫 화면에 크게 보이는 메인 영역. 브랜드 이름이나 슬로건이 들어가는 곳.

![lets do this](https://media.giphy.com/media/CjmvTCZf2U3p09Cn0h/giphy.gif)

## 왜 오로라였냐면

처음엔 그냥 단색 배경이었다. 어두운 배경에 텍스트. 깔끔하긴 한데 뭔가 심심했다.

레퍼런스 보다가 [rauno.me](https://rauno.me) 같은 사이트들에서 배경에 흐릿하게 빛이 번지는 효과를 봤다. 딱 이거다 싶었다. 오로라처럼 은은하게 번지는 색감 — 너무 화려하지 않으면서 살아있는 느낌.

색은 세 가지로 잡았다: 민트, 라벤더, 하늘색. 이 블로그 전체 컬러 톤이랑 맞게.

![히어로 섹션 완성본 — 데스크탑](/images/blog/hero-desktop.png)

## 구현 방식: 생각보다 단순하다

기술적으로는 `div` 세 개에 동그란 모양(border-radius: 50%)을 주고, 흐릿하게(blur) 만든 다음, 각자 다른 방향으로 움직이게 한 것.

> **CSS blur?** 포토샵의 흐림 효과와 같다. 숫자가 클수록 더 흐릿해진다.

포인트는 세 블롭이 **각자 다른 타이밍으로 움직이는 것**이었다. 같은 리듬으로 움직이면 규칙적으로 보여서 생동감이 없다. 애니메이션 시작 시점을 서로 엇갈리게 줬더니 훨씬 자연스러워졌다.

## 삽질 1: 블롭이 한쪽에서만 맴돌았다

처음에 움직임을 "200픽셀 오른쪽으로" 이런 식으로 고정 숫자로 줬다. 노트북에서는 괜찮았는데 데스크탑 큰 화면에서 보니까 블롭이 화면 왼쪽 구석에서만 왔다갔다 하고 있었다.

원인은 간단했다. 화면이 1400픽셀인데 200픽셀만 이동하면 전체 화면 기준으로는 거의 안 움직이는 거나 마찬가지였던 것.

해결은 고정 픽셀 대신 화면 크기 비율(vw/vh)로 바꾸는 것. 화면이 크면 더 많이 움직이고, 작으면 적게 움직이도록.

> **vw/vh?** 화면 크기를 100으로 나눈 단위. 1vw = 화면 너비의 1%. 화면 크기가 달라져도 비율은 유지된다.

## 삽질 2: 모바일에서 블롭이 사라졌다

데스크탑에서는 예쁘게 보이는데 모바일에서 확인하니 배경이 그냥 까맸다. 블롭이 없어진 게 아니라 잘리고 있었던 거였다.

히어로 섹션이 `max-width: 860px` 컨테이너 안에 들어가 있어서 양쪽이 뚝 잘렸던 것. CSS 트릭으로 히어로 섹션만 화면 꽉 차게 빼냈더니 해결됐다. *(기술 세부사항은 [오늘의 실수 모음](/blog/2026-03-10-mistakes-i-made-today) 참조)*

![히어로 섹션 — 모바일 (390×844)](/images/blog/hero-mobile.png)

## 타이틀 그라데이션

"기획자가"는 민트→흰색, "직접 만들기 시작했다"는 흰색→라벤더로 하고 싶었다.

근데 "기획자가"는 짧은 단어라 그라데이션을 걸면 흰색이 오른쪽 끝에서만 겨우 보였다. 색이 텍스트 전체에 걸쳐 서서히 바뀌어야 하는데, 짧은 글자에 긴 그라데이션을 끼워넣으니 끝부분 색이 거의 안 보이는 것.

중간에 색을 하나 더 추가해서 흰색이 더 일찍 등장하게 했다. 이런 건 코드 문제가 아니라 그냥 감각 문제다.

![타이틀 그라데이션 — 민트→흰색→라벤더](/images/blog/hero-title-gradient.png)

## 배운 것

CSS 애니메이션은 값 하나 바꾸는 것만으로 느낌이 완전히 달라진다.

투명도 0.5 vs 0.85, 흐림 효과 55px vs 70px, 속도 6초 vs 16초 — 이게 다 다른 분위기다. 코딩이라기보단 **감각 싸움**에 가까웠다.

기획자가 UI 스펙 쓸 때 "흐릿하게"라고 쓰면 개발자는 blur 값을 어떻게 잡을지 모른다. 이걸 직접 해보고 나니까 그게 얼마나 모호한 말이었는지 알게 됐다.
