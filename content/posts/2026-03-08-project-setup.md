# 개발 로그를 쓰면 블로그가 자동으로 올라오게 만들었다

직접 코딩을 시작하면서 제일 걱정됐던 게 기록이었다.

뭔가 배우고, 막히고, 해결하고, 넘어가면 — 다음 날이면 이미 흐릿해진다.
글을 쓰자니 개발보다 글쓰기가 더 오래 걸릴 것 같았고,
그렇다고 그냥 스쳐지나가긴 너무 아까웠다.

그래서 시스템을 먼저 만들었다. 개발하면서 메모만 하면
나머지는 Claude가 알아서 블로그 포스트로 만들어주는 구조.

## 어떻게 돌아가냐면

```
logs/dev-log.md 에 그날 한 일 짧게 메모
→ npm run generate:all 실행
→ 블로그 포스트 + 트위터 스레드 자동 생성
```

메모는 진짜 짧아도 된다. 오늘 뭘 했고, 어디서 막혔고, 어떻게 풀었는지.
그걸 Claude가 읽고 형식에 맞게 다듬어준다.

## 전체 구조

| 인풋 | 처리 | 아웃풋 |
|------|------|--------|
| `logs/dev-log.md` | Claude API | `content/posts/*.md` |
| `logs/dev-log.md` | Claude API | 트위터 스레드 |
| `content/posts/*.md` | Next.js | 블로그 페이지 |

## 만들면서 한 번 막혔던 것

YAML 날짜가 `Date` 객체로 파싱돼서 빌드가 터졌다.

```ts
// 이렇게 들어오는 게 아니라
date: "2026-03-08"

// 이렇게 들어왔음
date: 2026-03-08T00:00:00.000Z  // Date 객체
```

React가 Date 객체를 렌더링하려다 에러난 거였다.
`instanceof Date` 체크해서 `toISOString().substring(0, 10)` 으로 변환하니까 해결됐다.
별거 아닌데 이런 게 기록 안 해두면 다음번에 또 한참 찾게 된다.
