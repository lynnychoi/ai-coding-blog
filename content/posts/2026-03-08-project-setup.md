# 개발 로그를 쓰면 블로그가 자동으로 올라오게 만들었다

## 기록이 두려웠다

직접 코딩을 시작하면서 제일 걱정됐던 게 기록이었다.

뭔가 배우고, 막히고, 해결하고, 넘어가면 — 다음 날이면 이미 흐릿해진다. 글을 쓰자니 개발보다 글쓰기가 더 오래 걸릴 것 같았고, 그렇다고 그냥 스쳐지나가긴 너무 아까웠다.

그래서 시스템을 먼저 만들었다. 개발하면서 메모만 하면 나머지는 Claude가 알아서 블로그 포스트로 만들어주는 구조.

![automation magic](https://media.giphy.com/media/3oEjHZhG9COPG6XjzO/giphy.gif)

## 어떻게 돌아가냐면

```
logs/dev-log.md 에 그날 한 일 짧게 메모
→ npm run generate:all 실행
→ 블로그 포스트 + 트위터 스레드 자동 생성
```

> **npm run?** 프로젝트 안에 미리 만들어둔 명령어를 실행하는 것. 마치 세탁기 버튼처럼 — 내부 작동은 몰라도 버튼 하나로 돌아간다.

메모는 진짜 짧아도 된다. 오늘 뭘 했고, 어디서 막혔고, 어떻게 풀었는지. 그걸 Claude가 읽고 블로그 형식으로 다듬어준다.

## 전체 구조

| 인풋 | 처리 | 아웃풋 |
|------|------|--------|
| `logs/dev-log.md` (내 메모) | Claude API | `content/posts/*.md` (블로그 포스트) |
| `logs/dev-log.md` (내 메모) | Claude API | 트위터 스레드 |
| `content/posts/*.md` | Next.js (웹 프레임워크) | 블로그 페이지 |

## 만들면서 한 번 막혔던 것

날짜 형식 문제였다.

블로그 포스트 파일 맨 위에 이렇게 쓰면 (이 부분을 **YAML 프론트매터**라고 한다 — 마크다운 파일에 메타 정보를 붙이는 방식):

```
---
date: 2026-03-08
---
```

이걸 읽을 때 내가 원한 건 문자열 `"2026-03-08"` 이었는데, 코드가 자동으로 날짜 객체(`2026-03-08T00:00:00.000Z`)로 변환해버린 거다. 날짜를 "텍스트"가 아니라 "계산 가능한 날짜"로 인식한 것.

그래서 화면에 날짜를 표시하려고 하면 에러가 났다. 텍스트를 표시해야 하는 자리에 "날짜 데이터"가 들어온 거니까.

해결책은 "이게 날짜 객체이면 문자열로 바꿔서 써라"는 조건을 코드에 추가하는 것. 별거 아닌데 이런 게 기록 안 해두면 다음번에 또 한참 찾게 된다.
