# SD 카드에서 실수로 지운 아기 사진 32장, Claude Code로 복구하기

> Claude가 세션 직후 대신 기록. 특히 감정 섹션은 Lynn이 직접 다듬으면 더 좋음.

## 프로젝트 한 줄 소개

코딩 프로젝트가 아니다. 오늘 찍은 아기 사진을 SD 카드에서 실수로 삭제했고,
유료 복구 프로그램 없이 Claude Code랑 터미널만으로 전부 살려낸 기록.
"Claude Code는 개발 도구"라는 생각을 깨는 에피소드.

## 이 작업을 하게 된 이유

소니 카메라 SD 카드(128GB)에서 오늘 찍은 사진들을 실수로 지웠다.
지운 게 꽤 많았고, 그날 찍은 아기 사진이라 다시 찍을 수도 없는 것들.
Disk Drill, Wondershare Recoverit 같은 복구 프로그램을 먼저 다운받아 봤는데
— 스캔은 무료, 실제 복구는 유료(Disk Drill 기준 약 $89)였다.
혹시나 해서 Claude Code에 "SD 카드에서 이미지들을 실수로 삭제했는데 복구하고 싶어"라고 물었다.

## 뭘 했나 (작업 목록)

- Claude가 제일 먼저 한 말: **"지금부터 SD 카드에 아무것도 쓰지 마세요."**
  삭제된 파일은 '빈자리' 표시만 된 상태라 덮어쓰기 전엔 살릴 수 있다고.
- `diskutil list`로 SD 카드 찾기 (`/dev/disk10`, exFAT, 123.8GB).
  여기서 단서 발견: 카드 빈 공간이 927MB뿐 → 지워진 파일이 그 안에 다 있다는 뜻.
- 무료 도구 3개를 단계적으로 시도:
  1. **PhotoRec** (무료 카빙 도구) — 설치까지 했지만 파일명/날짜를 못 살리는 방식
  2. **TestDisk Undelete** — 삭제 파일 목록을 원본 이름+날짜로 보여줌. 터미널 TUI를 직접 조작해야 해서 중도 포기
  3. **The Sleuth Kit** (디지털 포렌식 도구) — 최종 선택. 스크립트로 전부 자동화 가능
- `fls` 명령으로 삭제된 파일 5,056개 목록을 원본 경로+날짜와 함께 추출
- 오늘 날짜(2026-06-06)로 필터 → **정확히 32장의 JPG**만 골라냄
- `icat` 명령으로 32장 전부 복구. 원본 파일명(DSC08241.JPG ~ DSC08275.JPG) 그대로
- 검증 3중: 파일 크기가 원본 메타데이터와 일치 / 32장 전부 유효한 JPEG / EXIF 촬영시각 보존 (오전 10:51 ~ 오후 13:45)
- Claude가 복구된 사진을 직접 열어서 확인 — 카시트에 앉은 아기 사진이 멀쩡하게 나왔다
- 마지막으로 친구한테 공유할 수 있는 쉬운 가이드 문서까지 정리 (`~/Desktop/SD카드-사진복구-가이드.md`)

## 막혔던 것들 / 삽질

- **sudo 비밀번호의 벽.** Claude는 관리자 비밀번호를 입력할 수 없어서 디스크를 직접 못 읽었다.
  Claude Code 안에서 `!` 프리픽스로 sudo 명령을 실행하는 것도 두 번이나 조용히 실패
  (비밀번호 프롬프트가 안 떠서 실행 자체가 안 된 듯).
  → 결국 **별도 터미널 앱에서 `sudo chmod o+r /dev/disk10*` 한 줄**만 내가 직접 실행.
  "읽기 권한만 잠깐 열어주면 나머지는 내가 다 할게" 방식으로 역할 분담이 정리됐다.
- **TestDisk TUI에서 길을 잃음.** Claude가 "①로그 화면에서 No Log, ②Intel 선택…" 하고
  단계별 표까지 만들어줬는데, 실제로는 안내에 없던 디스크 선택 화면이 떴다.
  화살표 키로 5,000개 파일 중 오늘 것만 고를 생각을 하니 아득해서
  "자동으로 오늘 날짜인 것들 너가 선택 도와주면 안될까..?"라고 물었고,
  여기서 도구가 Sleuth Kit으로 바뀌면서 모든 게 풀렸다.
- **Claude도 삽질했다.** 복구 스크립트 첫 실행이 32장 전부 실패.
  원인이 웃긴데 — zsh에서 `path`는 PATH 환경변수와 연결된 특수 변수라서,
  Claude가 반복문 변수 이름을 `path`로 짓는 바람에 셸의 PATH가 통째로 날아가
  `awk`, `ls` 같은 기본 명령어조차 "command not found"가 됐다.
  변수명을 `fp`로 바꾸니 32장 전부 성공. AI도 사람처럼 변수명으로 사고를 친다.
- 파일 번호 중간에 빠진 3장(DSC08248, 08249, 08262)은 카드 어디에도 없었다.
  촬영 직후 카메라에서 바로 지웠거나 카메라가 번호를 건너뛴 것으로 추정. 미해결로 남김.

## 결정의 이유 (왜 이렇게 만들었나)

- **유료 프로그램 대신 무료 CLI 도구:** Disk Drill $89 vs 무료. 같은 원리로 동작한다.
- **PhotoRec(전부 복구 후 필터) 대신 Sleuth Kit(선택 복구):**
  내가 "오늘 찍은 사진만 먼저 선택해서 복구하는 게 낫지 않을까?"라고 물은 게 분기점.
  PhotoRec은 파일명을 `f0012345.jpg` 같은 임의 이름으로 복구하지만,
  Sleuth Kit은 exFAT의 삭제된 디렉토리 항목을 읽어서 **원본 파일명과 날짜로 골라서** 복구할 수 있다.
- **복구 파일은 무조건 Mac 쪽에 저장:** SD 카드에 쓰면 다른 삭제 파일을 덮어쓸 수 있어서.
- **권한은 읽기만(o+r):** 쓰기 권한은 안 열었고, 카드를 뺐다 꽂으면 자동으로 원상복구되는 방식.

## 그때 어떤 기분이었나

(Lynn이 다듬을 것 — 세션에서 보인 흐름 기준으로 Claude가 초안)

지운 직후엔 유료 프로그램 결제 직전까지 갔다. 혹시나 하고 Claude Code에 물어본 거였는데,
"빈 공간이 927MB뿐이니 지워진 파일이 다 그 안에 있다"는 분석을 듣고 처음으로 살릴 수 있겠다 싶었다.
TestDisk 화면에서 안내에 없는 화면이 떴을 때가 제일 답답했고, "자동으로 해주면 안될까"라고
반쯤 포기하듯 물었는데 그게 오히려 정답이었다.
마지막에 Claude가 복구된 사진을 열어서 보여줬을 때 — 카시트에 앉은 아기 얼굴이
그대로 나왔을 때 — 그때 안도했다. 0원으로, 한 시간 안에, 32장 전부.

## 배운 것 / Takeaway

- **파일 삭제는 책을 버리는 게 아니라 도서관 목록 카드에 '폐기' 도장만 찍는 것.**
  덮어쓰기 전에 멈추면 살릴 수 있다. 그래서 첫 행동이 "카드 그만 쓰기"여야 한다.
- 유료 복구 프로그램이 하는 일을 무료 포렌식 도구(Sleuth Kit)가 그대로 한다.
  fls(목록 보기)와 icat(꺼내오기) 두 명령이 전부였다.
- AI와의 역할 분담: Claude는 비밀번호가 필요한 일을 못 한다.
  "권한 한 줄만 사람이 열어주고 나머지는 AI가" 패턴은 다른 작업에도 쓸 수 있겠다.
- 막히면 도구를 바꿔달라고 말해도 된다. "이거 너가 자동으로 해주면 안 돼?"라는
  비개발자스러운 질문이 더 좋은 기술적 선택(TUI → 스크립트)을 끌어냈다.
- AI도 변수명 하나로 삽질한다. 에러가 나도 원인을 설명받으면 무섭지 않다.
- **핵심 한 줄:** 복구 프로그램에 $89 내기 전에, Claude Code에 한 번 물어볼 것 — 지운 사진은 아직 거기 있다.
