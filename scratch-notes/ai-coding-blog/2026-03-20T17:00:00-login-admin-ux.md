# 로그인 삽질 총정리 — 엔터 한 줄이 만든 이틀치 고통

## 프로젝트 한 줄 소개
기획자/디자이너 출신 Lynn의 개인 블로그. Claude Code로 만든 개인 블로그인데, 글 쓰고 수정하는 어드민 페이지가 있고 비밀번호 로그인으로 보호돼 있다.

## 이 작업을 하게 된 이유
배포된 사이트에서 로그인이 자꾸 안 됐다. 로컬에서는 되는데 배포 환경에서만 실패.
그리고 로그인에 여러 번 실패하면 잠기는 기능을 만들었는데, 잠겼는지 아닌지 알 방법이 없었다.
게다가 로그인을 해도 다른 페이지로 이동하면 로그인이 유지되는 건지 아닌지 눈에 안 보였다.

## 뭘 했나 (작업 목록)
- 잠금 상태 표시 + 카운트다운 타이머 추가 (`app/admin/login/page.tsx`)
  - 잠긴 상태면 "🔒 N초 후 다시 시도할 수 있어요" 메시지
  - 폼 자체를 숨겨서 잠긴 동안 시도 자체를 막음
  - 로그인 페이지 진입할 때 자동으로 잠금 여부 체크
- 잠금 상태 확인 API 추가: `GET /api/admin/login`
- 잠금 시간 10분 → 2분으로 단축 (나 혼자 쓰는 어드민이라 과한 제한 필요 없음)
- 로그인 오류 메시지 수정: 서버 응답 메시지 그대로 표시하도록 (하드코딩 제거)
- 어드민 레이아웃 서버사이드 인증 추가 (`app/admin/layout.tsx`)
  - 서버 컴포넌트에서 쿠키 확인, 로그인 안 됐으면 `/admin/login`으로 리다이렉트
- AdminBar 컴포넌트 추가 (`app/components/AdminBar.tsx`)
  - 로그인 상태면 블로그 페이지에 항상 떠있는 플로팅 바
  - 글 상세 페이지에서는 "✏️ 편집" 버튼으로 바로 이동 가능
  - `/api/auth/status` 엔드포인트로 로그인 여부 확인
- 홈, 블로그 목록, 글 상세 페이지에 AdminBar 추가

## 막혔던 것들 / 삽질

### 배포 환경에서 비밀번호가 계속 틀렸던 이유
가장 황당한 버그. Vercel 환경변수에 `ADMIN_SECRET`을 추가할 때 `echo "비밀번호" | vercel env add` 방식으로 설정했는데, `echo`가 끝에 줄바꿈(`\n`)을 자동으로 붙여버렸다. 그래서 저장된 값이 `0110lynn\n`이 된 것.

로그인할 때 입력한 비밀번호는 `0110lynn`이고, 저장된 값은 `0110lynn\n`이니까 당연히 틀릴 수밖에 없다.

수정: `printf "비밀번호" | vercel env add ADMIN_SECRET production`으로 변경. `printf`는 줄바꿈을 안 붙인다.

이걸 찾는 데 꽤 걸렸다. 비밀번호가 맞는데 왜 틀리냐고 계속 의심했다.

### 로그인 오류 메시지 하드코딩
서버에서 응답 메시지를 보내줘도 클라이언트 코드에서 "비밀번호 틀렸어"를 하드코딩해서 무시하고 있었다. 서버에서 "잠금됐습니다" 라고 해도 항상 "비밀번호 틀렸어"로만 나왔다.
→ `data.error` 값을 그대로 표시하도록 수정.

### proxy.ts가 미들웨어로 인식 안 됨
어드민 보호를 위해 Next.js 미들웨어를 `proxy.ts`로 만들었는데 작동을 안 했다.
알고 보니 Next.js는 `middleware.ts`라는 파일명만 인식한다. 다른 이름은 그냥 일반 파일로 취급.
당장 고치는 대신 `app/admin/layout.tsx`를 서버 컴포넌트로 만들어서 쿠키 인증 처리. 이 방향이 더 깔끔했다.

## 결정의 이유 (왜 이렇게 만들었나)
- 잠금 시간 10분 → 2분: 나 혼자 쓰는 도구다. 10분은 너무 가혹했다.
- AdminBar 방식: 어드민 버튼을 공개 페이지에 두는 게 보안상 찜찜하다는 생각도 했는데, 쿠키 인증으로 서버에서 확인하고 로그인 상태일 때만 보이도록 하면 실질적인 위험은 없다고 판단.
- 서버사이드 레이아웃 인증: 클라이언트에서 리다이렉트하면 잠깐이라도 어드민 페이지가 노출될 수 있다. 서버 컴포넌트에서 처리하면 그럴 일이 없다.

## 그때 어떤 기분이었나
배포 환경에서 로그인이 안 될 때 진짜 당황스러웠다. 코드엔 문제가 없고 로컬에선 되는데, 왜? 하고 한참 뒤졌다.
`echo`가 줄바꿈을 붙인다는 걸 알았을 때는 좀 허탈하면서도 웃겼다. 이게 이런 거였어? 싶은 느낌.
어드민 바 만들고 나서 내 블로그에 들어갔을 때 편집 버튼이 뜨는 거 보고 뿌듯했다. 진짜 쓸 수 있는 툴이 된 것 같았다.

## 배운 것 / Takeaway
- `echo`는 줄바꿈을 붙인다. 환경변수에 값 넣을 때는 `printf` 써라.
- 배포 환경에서만 나는 버그는 환경변수를 먼저 의심해라.
- Next.js 미들웨어 파일명은 반드시 `middleware.ts`여야 한다. 다른 이름 안 됨.
- 서버 컴포넌트에서 인증 처리하면 클라이언트 리다이렉트보다 더 안전하고 깔끔하다.
- **핵심 한 줄**: 비밀번호가 맞는데 틀린다면, 비밀번호 끝에 줄바꿈이 붙어있을 수 있다.
